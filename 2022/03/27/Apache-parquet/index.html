Aapche Parquet

发表于 2022-03-27 更新于 2023-11-25 分类于 DataBase 本文字数： 1.7k 阅读时长 ≈ 2 分钟

原理

Parquet是一开源支持嵌套数据的列式存储格式 在大数据场景中 列存相对于行存能够提供更高的压缩比和更小的IO性能 可以只读需要的的数据降低,IO数据量,可以使用更高的压缩编码( Run Length Encoding 和 Delta Encoding 等)

Google 的 Dremel系统 通过使用 record shredding and assembly algorithm 来表达复杂的嵌套数据类型，同时将列进行高效压缩 Parquet就是基于Dremel的数据模型和算法的工程实现.

Parquet是不绑定数据处理框架的的 可以适配多种组建查询引擎、计算框架、数据模型 parquet-format 定义了 Parquet内部的数据类型和存储格式 parquet-mr 包含了外部对象结构到Parquet数据类型的映射和转换

存储模型

Parquet的存储模型 由 Row Group、Column Chunk、 Page 组成 RowGroup 水平方向将数据划分为行组 默认于HDFS BLOCK 块对齐 一般为1G 减少对HDFS集群namenode造成压力，同时大的RowGroup 可以带来比较大的并行度, Column Chunk 行组中的每一列保存在一个列块中 Page 一个列块包含多个页 页是最小的编码单位
